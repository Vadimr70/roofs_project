Суть механизма поиска.
Поиск состоял в фильтрации строений с плоской крышей с площадью не менее 1000 кв. м . Важно, что в разметке игнорировались строения с
металлическими крышами т.е. даже если на крыше нет четко различимого конька, но сама крыша состоит из нескольких металлических блоков
(визуально) или имеет другие “визуальные” следы металлических пластин, то такая крыша не выделялась.

После обучения, нейросеть применялась для получения масок снимков, другими словами, сеть сама выделяла нужные крыши. На след. этапе полученные результаты обрабатывались - удалялся шум, выравнивались неточности в масках крыш.
Далее применялся поиск контуров, центры которых в последствие стали координатами предсказанных строений.

Для запуска необходимо запустить скрипт ./process.sh с указанием директории, в которой лежат карты, скачанные плагином QTiles. 
Важно: должен быть указан относительный путь. Рекомендуется создать папку, например с именем example в самом репозитории. В этой папке должны располагаться все корневые папки карт (у каждой такой папки имеется подпапка "16").

./process.sh --process_dir

В случае с примером выше:

./process.sh example 

В результате работы программы в локальном репозитории будет создана папка outputs с визуализацией работы. Итоговая таблица будет иметь название Results.xlsx и находиться в репозитории.
Важно: нейросеть требует определенный формат данных, который генерируется на первом этапе работы скрипта. Для сохранности, оригиналы изображений удаляться не будут, а будет лишь создаваться копия данных в нужном для сети формате. В связи с этим на диске должно иметься как минимум 80% свободного места от размера изначальных данных.
